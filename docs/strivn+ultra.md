## The Strivn Baseline Standards

Before a single line of code is written on any project—whether it is a local portfolio or a massive SaaS platform—we adhere to the following non-negotiable standards:

### 1. Intentional Architecture
- **Stack Selection:** Technologies are chosen based on the problem's scale, not hype. (e.g., Next.js for high-SEO SaaS, Vanilla + Supabase for lightweight ultra-fast portals).
- **Pre-Flight Mapping:** Database schemas, routing flows, and security measures (RLS) are mapped and approved *before* coding begins.

### 2. Enterprise-Grade Telemetry
No project ships blind. Every product must have built-in eyes and ears:
- **Analytics:** Google Analytics 4 (GA4) / Google Search Console (GSC) for traffic and SEO mapping.
- **Session Tracking:** PostHog for analyzing user workflows and UI friction.
- **Crash Reporting:** Sentry for zero-latency bug tracking and error catching.
- **Bot Protection:** Cloudflare Turnstile to prevent spam and abuse automatically.

### 3. Bulletproof Security & Performance
- **Zero-Trust Backend:** Row Level Security (RLS) and strict authenticated policies via Supabase or equivalent. 
- **Lighthouse Scores:** Assets must be optimized, heavily compressed, and cached. We do not accept slow load times.
- **Flawless Responsiveness:** The UI must feel completely native on Mobile, Tablet, and Desktop.

### 4. Semantic SEO Mastery
- We do not rely on "hope." We inject dynamic Schema.org structured data natively into the HTML to ensure Google and other search engines explicitly understand what we are serving.
